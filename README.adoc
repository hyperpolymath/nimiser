== What Is This?

Nimiser generates **high-performance C libraries** from high-level
descriptions using Nim's extraordinary metaprogramming system.

Nim compiles to C with Python-like syntax and macros that make Rust's
look primitive. Nimiser uses this power for library generation.

== How It Works

Describe your library interface in `nimiser.toml`. Nimiser:

1. Generates Nim code with compile-time metaprogramming
2. Compiles Nim to optimised C (or C++, Objective-C, JavaScript)
3. Produces a standalone C library with clean headers
4. Creates a Zig FFI bridge for integration

== Key Value

* **High-level descriptions → optimised C libraries**
* **Nim's metaprogramming** generates specialised code paths
* **Multi-backend** — C, C++, JavaScript from one source

== Architecture

Follows the hyperpolymath -iser pattern (same as https://github.com/hyperpolymath/chapeliser[Chapeliser]):

* **Manifest** (`nimiser.toml`) — describe WHAT you want
* **Idris2 ABI** (`src/abi/`) — formal proofs of interface correctness
* **Zig FFI** (`ffi/zig/`) — C-ABI bridge to target language runtime
* **Codegen** (`src/codegen/`) — generates target language wrapper code
* **Rust CLI** — parses manifest, validates, generates, builds, runs

User writes zero target language code. Nimiser generates everything.

Part of the https://github.com/hyperpolymath/iseriser[-iser family] of acceleration frameworks.
